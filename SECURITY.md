# Security policy

LOOP is testnet-only. Do not send mainnet assets to the published testnet contract or rely on it as independently audited software.

Report vulnerabilities through [GitHub private vulnerability reporting](https://github.com/rub1kub/loop/security/advisories/new). Include a minimal impact description and sanitized reproduction. Never send wallet mnemonics, bot tokens, Telegram `initData`, wallet proofs or personal data.

Public issues containing active credentials or exploit details may be removed to protect users. The maintainer will acknowledge actionable reports and coordinate remediation and disclosure.
